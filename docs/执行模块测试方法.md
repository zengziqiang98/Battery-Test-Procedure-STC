# 执行模块测试方法

## 1. 推荐测试层级

执行模块建议分三层测试：

1. 单板上电检查：确认地址脚、电源、ADC、控制输出默认安全。
2. 通信检查：通过中转模块 IIC 链路或执行模块 UART1 Modbus 直连读写寄存器。
3. 带负载动作检查：接电池或假负载，验证充电、放电、完成、停止、异常保护。

## 2. 不拆系统测试

正式链路是：

`Python 上位机 -> 串口 -> 中转模块 -> IIC -> 执行模块`

优先用上位机测试：

- 扫描模块，确认地址在线。
- 读电压、充电电流、放电电流、充电容量、放电容量。
- 发 `START_CHARGE`，确认充电控制脚动作、状态位为正在充电。
- 发 `STOP_CHARGE`，确认控制脚关闭。
- 发 `START_DISCHARGE`，确认放电控制脚和风扇动作、状态位为正在放电。
- 发 `STOP_DISCHARGE`，确认控制脚关闭。

长测前至少连续轮询 10 分钟，确认没有离线、容量回退、状态位乱跳。

## 3. 单板 UART1 直连测试

当前执行模块保留了 UART1 Modbus RTU：

- UART1 引脚：P3.0 RXD / P3.1 TXD
- 波特率：256000
- 地址：硬件地址脚读出后 `+1`

UART1 用于命令/寄存器直连测试，不建议同时输出文本日志，否则会污染 Modbus 帧。

常用寄存器：

- `0x0000` 写命令：`1` 开始放电，`2` 停止放电，`3` 开始充电，`4` 停止充电，`5` 急停
- `0x0001` 充电状态位
- `0x0002` 放电状态位
- `0x0003` 充电容量 mAh
- `0x0004` 放电容量 mAh
- `0x0005` 电池电压 mV
- `0x0006` 充电电流 mA
- `0x0007` 放电电流 mA
- `0x000B` 设备 ID

## 4. 可选 UART2 调试口

代码已加入可编译开关：

```c
#define EXECUTOR_DEBUG_UART 0
```

需要内部状态日志时，把它改成 `1` 后重新编译烧录。

调试口配置：

- UART2_TX：P4.6
- UART2_RX：P4.7
- 波特率：256000
- 默认只需要接 TX 和 GND

输出示例：

```text
DBG BOOT
DBG A=1 S=0 OC=0 OD=0 V=12345 IC=0 ID=0 QC=0 QD=0
```

字段含义：

- `A`：模块地址
- `S`：内部状态，`0` 空闲，`1` 充电，`2` 放电，`3` 充电完成，`4` 放电完成
- `OC`：充电状态位
- `OD`：放电状态位
- `V`：电池电压 mV
- `IC`：充电电流 mA
- `ID`：放电电流 mA
- `QC`：充电容量 mAh
- `QD`：放电容量 mAh

正式长测固件建议保持 `EXECUTOR_DEBUG_UART 0`，避免额外串口输出影响时序。

## 5. 长测前检查清单

- 空闲上电后充放电控制脚必须为关闭。
- 急停命令能同时关闭充电、放电和风扇。
- 开始充电后状态位应包含正在充电，电流/容量应增长。
- 停止充电后控制脚关闭，状态不应继续显示正在充电。
- 开始放电后状态位应包含正在放电，风扇打开，放电容量应增长。
- 停止放电后放电控制脚和风扇关闭。
- 充电完成位和放电完成位会保持到上位机发送 STOP，不应只闪一下。
- 拔掉电池或制造无电池条件时，上位机应判该模块 NG，不应卡住整批测试。
