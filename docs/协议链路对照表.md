# 协议链路对照表

## 目标拓扑

Python 上位机通过串口总线连接多个 STC 中转模块。每个中转模块有自己的总线地址，并通过 IIC 管理本通道配套的 STC 执行模块。

```text
Python UI -> 串口协议 -> STC 中转模块 N -> IIC -> STC 执行模块
```

## Python 到中转模块串口协议

当前使用 `python_app/protocol_thread.py`，不是 `测试.py` 里旧的 `0xAA` 帧协议。

请求帧：

```text
[ADDR][CMD][LEN][DATA...][XOR]
```

响应帧：

```text
[ADDR][CMD|0x80][STATUS][LEN][DATA...][XOR]
```

字段：

| 字段 | 含义 |
| --- | --- |
| ADDR | 中转模块地址，1-64；0 表示广播 |
| CMD | 串口协议命令 |
| LEN | DATA 字节数 |
| STATUS | 0x00 成功，0xFF 失败 |
| XOR | 前面所有字节异或 |

Python 轮询帧：

```text
[addr][0x08][0x03][0x00][0x01][0x0B][xor]
```

含义：从寄存器 `0x0001` 开始读 `0x0B` 个 16 位寄存器。Python 期望响应总长 27 字节，数据区 22 字节。

## 中转模块到执行模块 IIC 协议

执行模块当前 IIC 从机命令定义在 `firmware/stc8h/src/i2c_cmd.h`，从机响应数据末尾追加 1 字节 XOR 校验。

写命令：

| IIC 命令 | 数据 | 说明 |
| --- | --- | --- |
| 0x01 | 无 | 开始放电 |
| 0x02 | 无 | 停止放电 |
| 0x03 | 无 | 开始充电 |
| 0x04 | 无 | 停止充电 |
| 0x07 | 2 字节 | 设置过充保护电压 |
| 0x08 | 2 字节 | 设置过放保护电压 |
| 0x0E | 1 字节 | 设置循环放电次数 |
| 0x24 | 1 字节 | 设置充电电压档位 |
| 0x25 | 1 字节 | 设置充电电流 |
| 0x27 | 1 字节 | 设置电池类型 |

读命令：

| IIC 命令 | 返回 | 对应串口寄存器 |
| --- | --- | --- |
| 0x05 | 1 字节 + XOR | STATUS_CHARGE |
| 0x06 | 1 字节 + XOR | STATUS_DISCHARGE |
| 0x09 | 2 字节 + XOR | CHARGE_MAH |
| 0x0A | 2 字节 + XOR | DISCHARGE_MAH |
| 0x0B | 2 字节 + XOR | BAT_VOLT |
| 0x0C | 2 字节 + XOR | OVERC_PROT |
| 0x0D | 2 字节 + XOR | OVERD_PROT |
| 0x0F | 1 字节 + XOR | CYCLE_NUM |
| 0x10-0x19 | 2 字节 + XOR | CYCLE_DISCH_END |
| 0x1A-0x23 | 2 字节 + XOR | CYCLE_CHG_END |
| 0x26 | 2 字节 + XOR | BAT_VOLT |
| 0x28 | 2 字节 + XOR | CHARGE_CURRENT |
| 0x29 | 2 字节 + XOR | DISCHARGE_CURRENT |

状态位转换：

| 执行模块状态位 | Python 合成 status |
| --- | --- |
| charge status bit2 | 0x01 正在充电 |
| discharge status bit2 | 0x02 正在放电 |
| charge status bit0 | 0x04 过充报警 |
| discharge status bit0 | 0x08 过放报警 |
| charge status bit1 | 0x10 充电完成 |
| discharge status bit1 | 0x20 放电完成 |
| charge status bit3 | 0x40 无电池 |

## 中转模块缓存策略

中转模块不再在串口高速轮询时同步读取 IIC。`firmware/stc8h/bridge/protocol.c` 现在把上位机高频批量读的 `0x0001-0x000B` 寄存器放入缓存：

| 行为 | 说明 |
| --- | --- |
| 串口读 `0x0001-0x000A` | 直接返回缓存值；缓存无效时快速返回 `STATUS=0xFF` 和 0 值 |
| 串口读 `0x000B` | 中转本地返回 `DEVICE_ID=0x0101`，不访问 IIC |
| 后台刷新 | 主循环空闲时每次最多读 1 个 IIC 寄存器，轮询刷新 `0x0001-0x000A` |
| 执行模块离线 | 连续失败后清除非设备 ID 缓存，并进入短冷却，避免每次串口轮询都被 IIC 超时拖慢 |
| 写命令 | 仍实时转发到执行模块 IIC |

`0x0020-0x0039` 循环历史容量目前仍保留现场 IIC 读取，不应放在高速轮询路径中。
